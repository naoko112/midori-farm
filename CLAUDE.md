## 制約事項

- GitHub の main ブランチへの push = releaseである
  - release の前にかならず確認すること
- publicリポジトリであるため、個人情報やクレデンシャル情報が入力された場合は release を中止する。
- わからない部分は勝手に補完せず、問い合わせを行う
- 会話の中で確定した要件は requirements.md に記載する
  - 新しく出てきた要件によって過去の要件に修正が生じた場合は問い合わせを実施する。
